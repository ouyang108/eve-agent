# 日志分析报告 — codecare-agent 项目

## 1. 日志系统架构

项目使用**自定义审计日志系统**，定义在 `agent/lib/audit.ts` 中。

- **日志文件**: `.codecare/audit.log`（JSONL 格式，每行一个 JSON 记录）
- **写入方式**: `appendAudit()` 异步追加
- **读取方式**: `readAudit()` 解析返回 `AudioRecord[]`
- **记录结构**:
  ```ts
  interface AudioRecord {
    type: string;       // 操作类型
    detail: unknown;    // 操作详情
    createdAt?: string; // 时间戳（ISO格式）
  }
  ```

## 2. 日志记录点

以下工具/模块调用了 `appendAudit()` 写入 audit.log：

| 来源文件 | type 值 | 记录内容 |
|---|---|---|
| `agent/tools/inspect_project.ts` | `inspect_project` | facts, data (项目元信息) |
| `agent/tools/run_safe_command.ts` | `tool.run_safe_command` | command, cwd, exitCode |
| `agent/tools/run_safe_command.ts` | `tool.run_safe_command.blocked` | command, reason (被阻止的命令) |
| 通过工具调用记录的其他类型 | `tool.read_workspace_file` | path, bytes |
| | `tool.write_workspace_file` | path, reason, backupId, bytes |
| | `tool.update_task` | id, title, status, risk |
| | `tool.search_text` | query, path, hasMatches |
| | `tool.remember_fact` | scope, key, confidence |

## 3. 搜索结果：无 console.* 日志

在整个 TypeScript/JSON/Markdown 源码中，**没有发现任何 `console.log`、`console.warn`、`console.error`、`console.info`、`console.debug`、`console.trace` 调用**。
